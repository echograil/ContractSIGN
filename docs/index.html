ContractSIGN v0.2

一个面向英文合同的本地优先问答流水线：上传 PDF、输入中文或英文问题，系统按六个模块搬运文件、检索条款、生成带依据的回答，并保留观测数据。

查看 README 本地 UI 源文件

当前可用体验

上传合同并提问本地 Web UI 支持 PDF 上传、问题输入、API 开关和结果展示。
可视化流水线页面展示六个工位：接入、检索、路由、生成、风险依据、观测导出。
文件搬运可见每个模块都有独立 input/ 和 output/，运行后可直接查看产物。
中文问题支持本地检索内置中文合同问题到英文法律词的扩展层，常见摘要、付款、终止、保密、违约等问题可用。
可复制结果本地 UI 支持复制回答、复制运行摘要、下载完整运行 JSON 报告。
可选 API 增强默认可离线跑；打开 API 路由、Embeddings 或生成后，可用自己的 API Key 增强效果。

GitHub Pages 能做什么

GitHub Pages 是静态站点，只能运行浏览器里的 HTML/CSS/JS。它不能在 GitHub 服务器上接收 PDF、执行 Python、写入模块文件夹，也不能完成当前这种 input/ 到 output/ 的文件搬运流水线。

因此推荐分工如下：

  • GitHub Pages：展示项目介绍、当前体验、开发进度、安装说明和演示截图。
  • 本地 Web 服务：运行真实 PDF 上传、问题回答、文件搬运、SQLite 观测。
  • 如果需要云端在线处理：另接后端服务，例如 FastAPI、serverless 或容器服务。

本地启动

python -m pip install -r requirements.txt
python scripts/web_server.py

打开 http://127.0.0.1:8765。如需 API 功能，复制 .env.example 为 .env 并安装 requirements-api.txt。

当前边界

  • 本地中文检索是词典扩展和兜底，不等同于完整跨语言语义理解。
  • 复杂中文改写和深层法律推理建议启用 API Embeddings 与 API 生成。
  • 风险模块目前偏规则提示，事实问答场景下提示语可能略重。
  • 扫描版或图片型 PDF 仍可能无法接入，需要 OCR 扩展。
